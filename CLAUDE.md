# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

The Featureflow documentation site for [docs.featureflow.io](https://docs.featureflow.io), a
[Docusaurus 3](https://docusaurus.io/) static site. Content is Markdown/MDX; there is very little
application code. Node version is pinned in `.nvmrc` (21.7.3); package manager is Yarn with
`nodeLinker: node-modules` (`.yarnrc.yml`).

## Commands

```bash
yarn                 # install
yarn start           # local dev server with live reload
yarn build           # build static site into build/ (also the real lint — see below)
yarn serve           # serve a production build locally
yarn clear           # clear the Docusaurus cache (.docusaurus/) when things go stale
```

Deployment pushes to GitHub Pages: `GIT_USER=<github-user> yarn deploy`.

There is no separate test or lint step. `yarn build` is the gate: `onBrokenLinks: 'throw'` in
`docusaurus.config.js` means any broken internal link fails the build, so run `yarn build` to
validate content changes.

## Content structure & conventions

- Docs are served at the site **root** (`routeBasePath: '/'`), not under `/docs`.
- The sidebar is **autogenerated from the folder structure** (`sidebars.js`). Ordering is
  controlled by the numeric prefixes on folders and files (e.g. `01-Quickstart/`,
  `1-quickstart-server.md`). Category labels come from `_category_.json` files in each folder.
- `sidebars.js` also appends a manual "SDKs on Github" category of external links. SDK source
  code lives in separate `featureflow/*` repos, not here — this repo only documents them.

## Versioning (important)

This site uses Docusaurus versioned docs. `versions.json` lists released versions (`2.0.0`,
`1.0.0`).

- `docs/` is the **current / in-progress** version (the "next" docs).
- `versioned_docs/version-X.Y.Z/` holds **frozen snapshots** of released versions, with matching
  `versioned_sidebars/`.

**`2.0.0` is the default/current published version** — it's what visitors see at
https://docs.featureflow.io/ and the version most edits target. Its content lives in
`versioned_docs/version-2.0.0/`, **not** in `docs/`. Editing files under `docs/` changes only the
in-progress "next" docs, which is not what most visitors see; to change the live site, edit the
corresponding file under `versioned_docs/version-2.0.0/`. To cut a new version:
`yarn docusaurus docs:version X.Y.Z`.

## OpenAPI / REST API docs

Swagger/OpenAPI is **not** rendered in this site — it did not display well here. Instead the REST
reference is served by a separately hosted Swagger UI (from the Featureflow Java project) at
https://api.featureflow.com/api/swagger-ui/index.html, which the navbar "API Reference" link
points to. The root `openapi.json` and the installed `docusaurus-plugin-openapi-docs` /
`docusaurus-theme-openapi-docs` packages are not wired into `docusaurus.config.js`. The
hand-authored REST content here is `rest-responses.mdx` (in both `docs/` and the versioned trees).

## Custom code

Minimal. `src/components/HomepageFeatures/` is the landing-page feature grid and
`src/css/custom.css` holds theme overrides. Sass is available (`docusaurus-plugin-sass`).
